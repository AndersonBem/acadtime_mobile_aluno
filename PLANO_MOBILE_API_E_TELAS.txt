PLANO MOBILE - API, BACKEND, ROTAS E TELAS

Contexto
- App mobile focado no aluno.
- Login atual usa /login/ e ja funciona com bloqueio no mobile para permitir apenas aluno.
- Pasta web frontend nao entra neste fluxo.
- Foto de perfil usa endpoint mobile proprio e salva arquivo na AWS, seguindo estrategia parecida com certificados.


ARQUIVOS DE API MOBILE JA CRIADOS

- src/api/client.js
  Centraliza URL da API, token em memoria, headers e chamadas fetch.

- src/api/tokenStorage.js
  Salva e restaura token/usuario usando expo-secure-store.

- src/api/auth.js
  Login, restauracao de sessao, logout e recuperacao de senha.

- src/api/aluno.js
  Busca perfil do aluno logado e cursos vinculados.

- src/api/submissoes.js
  Lista, detalha, resume e cria submissoes. Tambem monta URL do certificado.

- src/api/atividades.js
  Lista tipos de atividade e cria atividade complementar.

- src/api/ocr.js
  Envia certificado para extracao OCR.

- src/api/cursos.js
  Lista inscricoes/cursos do aluno.

- src/api/dashboard.js
  Busca dados do dashboard mobile.

- src/api/notificacoes.js
  Lista notificacoes do aluno, marca uma como lida, marca todas como lidas e filtra lidas/nao lidas no app.

- src/api/perfil.js
  Busca perfil do aluno e atualiza foto de perfil.

ENDPOINTS NOVOS CRIADOS

1. GET /mobile/dashboard/
   - Retorna aluno, curso atual, cursos, resumo de horas, progresso por tipo e ultimas submissoes.
   - Usado pela tela Dashboard.

2. GET /mobile/notificacoes/
   - Retorna notificacoes do aluno derivadas das submissoes.
   - Usa persistencia em NotificacaoMobileLeitura para informar lida true/false.

3. PATCH /mobile/notificacoes/{id}/marcar-lida/
   - Marca uma notificacao especifica como lida.
   - Persiste leitura por aluno em NotificacaoMobileLeitura.

4. POST /mobile/notificacoes/marcar-todas-lidas/
   - Marca todas as notificacoes do aluno como lidas.
   - Persiste leitura por aluno em NotificacaoMobileLeitura.

5. GET /mobile/perfil/
   - Retorna dados do perfil do aluno para a tela de perfil.
   - Retorna foto_perfil_url quando existir.

6. PATCH /mobile/perfil/foto/
   - Atualiza foto de perfil do aluno.
   - Salva arquivo na AWS via default_storage.
   - Salva URL e metadados na tabela AlunoFotoPerfil.


ENDPOINTS FUTUROS/OPCIONAIS

1. POST /login-aluno/
   - Fica para depois.
   - Login exclusivo para aluno no backend.
   - Hoje nao e prioridade porque o app ja bloqueia coordenador/superadmin depois do /login/.

2. GET /mobile/cursos/
   - Opcional.
   - Pode listar cursos/inscricoes do aluno sem depender de /inscricao/.
   - So criar se a tela de nova submissao precisar.

3. GET /mobile/cursos/{id}/regras-atividade/
   - Opcional.
   - Pode listar regras de atividade do curso para o mobile sem alterar RegraAtividadeViewSet.
   - So criar se a tela de nova submissao precisar.


TABELAS/MODELS NOVOS

1. NotificacaoMobileLeitura
   - Salva quais notificacoes foram lidas por aluno.
   - Relaciona aluno, submissao e status_submissao.
   - Usada por:
     - GET /mobile/notificacoes/
     - PATCH /mobile/notificacoes/{id}/marcar-lida/
     - POST /mobile/notificacoes/marcar-todas-lidas/

2. AlunoFotoPerfil
   - Salva foto de perfil do aluno.
   - Campos principais:
     - aluno
     - nome_arquivo
     - url_arquivo
     - data_upload
   - Arquivo fica na AWS.

   - Nao mexer agora para evitar risco na parte web.

4. POST /login-aluno/
   - Criar endpoint de login exclusivo para aluno se sobrar tempo.
   - Hoje o app bloqueia login de coordenador/superadmin no auth.js.

O QUE FALTA NO MOBILE PARA DASHBOARD E DETALHES

1. Instalar e configurar React Navigation.
2. Criar pasta routes/router.
3. Restaurar sessao no carregamento inicial do app.
4. Redirecionar para AuthRoutes ou AppRoutes.
5. Conectar Dashboard com getDashboardAluno().
6. Criar estados de loading, erro e vazio no Dashboard.
7. Criar tela SubmissionDetails.
8. Navegar do card de submissao para detalhes passando id_submissao.
9. Conectar SubmissionDetails com getSubmissaoPorId(id).
10. Implementar abrir/baixar certificado.
    - Primeira versao pode usar Linking.openURL(url).


OBSERVACOES IMPORTANTES

- Endpoints de notificacao lida ja usam persistencia em NotificacaoMobileLeitura.
- Foto de perfil ja usa AWS/storage e tabela AlunoFotoPerfil.
- Login aluno exclusivo no backend fica para depois.
- O mobile deve continuar tratando somente usuario tipo aluno.
- Evitar alterar endpoints compartilhados com a web quando um endpoint mobile isolado resolver.
